## Practical Development Notes

- Keep your project folders organized inside the local web directory so each xampp server project has a clear URL and file structure.
- Use xampp phpmyadmin to create separate databases for each application instead of sharing one database across unrelated tests.
- When using xampp wordpress, match the local PHP version as closely as possible to the server where the site will eventually run.
- If you plan to download xampp for windows on a new machine, confirm that ports 80 and 443 are not already reserved by another service.
- Treat localhost xampp as a development environment only, and avoid exposing unfinished xampp mysql projects directly to the public internet.

## Fixing Common XAMPP Problems

- Apache will not start? Check whether another app is already using port 80 or 443, then adjust the port or close the conflicting service before restarting apache xampp.
- Database connection failed? Confirm MariaDB is running, review credentials, and verify the application points to the correct xampp mysql database.
- phpMyAdmin will not open? Restart the local services, clear the browser cache, and open xampp phpmyadmin from the control panel link.
- WordPress setup is stuck? Recheck the database name, username, and local URL when configuring xampp wordpress on localhost xampp.
- Installer issues on Windows? Run the setup as administrator, choose a simple folder path, and use xampp download windows only from the official source.
